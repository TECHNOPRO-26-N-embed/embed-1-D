## 4. 一番苦労したこと、どう乗り越えたか（30秒）

<!-- ここが発表の山場です。1つだけに絞って、ストーリーで書いてください -->

**何が起きたか：**実機テストの際、センサーの前に対象物を置いても、測定値が常時 ⁠135cm〜150cm⁠ の固定値から動かなくなる現象が発生した。さらに、センサー自体をブレッドボードから完全に抜いた状態でも、同じ固定値がシリアルモニターに出力され続けるという不可解な挙動に直面した。


**どう対処したか：**「センサーを抜いても数値が出る」という事実から、Arduino の Echo ピンが物理的に未導通になり、タイムアウトを起こしていると仮定した。


**そこから何がわかったか：**そこで、ブレッドボードの内部端子の接触不良を疑い、ブレッドボードを完全にバイパスして、ジャンパ線で Arduino とセンサーを直接結線した。その結果、信号の導通が完全に回復し、正確な距離測定に成功した。

## 6. 発表で見せたいもの（メモ）

<!-- 動くデモ、配線の写真、設計図、フローチャート、AIとのやり取りのスクショなど -->
<!-- 動くものがなくても、「考えた過程」を見せられれば十分です -->
**【考えた過程】データフリーズ原因の特定から解決までのプロセス**
  1. 問題の発見： 距離データが「150cm」の固定値から動かなくなり、センサーを抜いても値が変わらないという不可解な挙動に直面。
  2. 仮説の構築： AIに状況を共有してディスカッションを行い、「受信ピン（Echo）が物理的にどこにも繋がっていない『未導通（フローティング）』状態になり、空間の電磁ノイズを誤検知してタイムアウトを起こしているのではないか」という仮説を立てました。
  3. 検証と解決：この仮説を元に回路を物理的に確認したところ、ブレッドボードの穴の緩みによる接触不良を発見。ジャンパ線で直接結線（直連）にバイパスすることで、正常にデータを取得できるようになりました。
