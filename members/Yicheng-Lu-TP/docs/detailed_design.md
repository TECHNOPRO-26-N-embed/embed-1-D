### `loop()` — メインループ

> ※ loop() は「状態ごとに何をするか」だけ書く。細かい処理は各関数に任せる。

```
【処理の流れ】

＜毎ループ実行すること＞
  - 入力を読む（readButton(), readSensor() などを呼ぶ）
  - 現在時刻を取得: now = millis()

＜currentState が 0（待機中）のとき＞
  - センサー値を監視する
  - 検知条件を満たしたら → currentState = 1

＜currentState が 1（動作中）のとき＞
  - メイン処理を行う
  - 終了条件を満たしたら → currentState = 2

＜currentState が 2（完了）のとき＞
  - 完了表示をする
  - リセットボタンが押されたら → currentState = 0

＜currentState が 3（エラー）のとき＞
  - エラー表示をする / リセットを待つ
```

**↓ 自分の loop() を設計してください**
```
【処理の流れ】

1. 距離計測関数の呼び出し
   - readDistance() を実行して最新の距離を取得
2. 状態更新関数の呼び出し
   - updateState() を実行して現在の計測値から状態を判定
3. 各アクチュエータの制御関数呼び出し
   - controlMotor() を実行してモーター駆動
   - controlBuzzer() を実行してブザー鳴動制御

### `関数名()` — （役割を1行で書く）

### `readDistance()` - (距離データの計測)
**basic_design.md 2-2 との対応：** 超音波センサーを制御し距離を算出して変数に格納する。
- **引数：** なし
- **戻り値：** void
【処理の流れ】
1. PIN_TRIG を 2μs ほど LOW にしたのち、10μs 間 HIGH にして超音波を発信。
2. pulseIn(PIN_ECHO, HIGH) を用いて音波が反射して戻るまでの時間（μs）を計測。
3. 計測された時間から、距離（cm）を「時間 / 58」の計算式で算出し、変数 `sensorValue` に格納。

【エラー・異常ケース】
- 測定エラーで `sensorValue == 0` または `400` 以上の異常値が来た場合：
  - 値を更新せず、前回の正常な値をキープしてスキップする。
### `updateState()` - (状態遷移の判定)
**basic_design.md 2-2 との対応：** 距離値をもとに走行状態（3段階）を決定する。
- **引数：** なし
- **戻り値：** void

【処理の流れ】
1. もし `sensorValue` が 5cm 以下のとき：
   - `currentState` を 2 (緊急停止) に更新。
2. もし `sensorValue` が 5cm 超 かつ 15cm 以下のとき：
   - 現在の状態が「2（緊急停止）」でなければ、`currentState` を 1 (減速) に更新。
   - ※安全要件（オミット方針）に基づき、一度状態2に入った場合は自動復帰させない。
3. もし `sensorValue` が 15cm 超のとき：
   - 現在の状態が「2（紧急停止）」でなければ、`currentState` を 0 (巡航) に更新。
【エラー・异常ケース】
- 15cm や 5cm などの境界付近で値が小刻みに揺れた場合（ハンチング）：
  - 条件判定に ±1cm の猶予（ヒステリシス）を持たせ、状態がガタガタ変わるのを防ぐ。
### `controlMotor()` - (パルス制御による調速走行)
**basic_design.md 2-2 との対応：** 状態に応じてステッピングモーターの速度をパルス幅で制御。
- **引数：** なし
- **戻り値：** void
【処理の流れ】
1. `millis()` を呼び出し、現在の時刻を `now` に取得。
2. `currentState` の値に応じて、ターゲットとなるパルス間隔（`targetInterval`）を分岐。
   - 状態0 (巡航) → `targetInterval = INTERVAL_NORMAL (2ms)`
   - 状態1 (減速) → `targetInterval = INTERVAL_SLOW (5ms)`
   - 状態2 (停止) → 各モーターピンを全て LOW にして関数を抜ける
3. 「`now - lastMillis >= targetInterval`」が成立するか判定。
   - 成立した場合：ステッピングモーターの励磁ステップを1進め（4相1-2相励磁など）、`lastMillis = now` で時刻を更新。
   - 成立しない場合：処理をブロック（delay）せずにそのまま関数を抜ける（ノンブロッキング処理）。
### `controlBuzzer()` - (警告ブザーの即時鳴動)
**basic_design.md 2-2 との対応：** 緊急停止状態のときのみブザーを鳴らす。
- **引数：** なし
- **戻り値：** void



【処理の流れ】
1. もし `currentState == 2`（緊急停止）の場合：
   - PIN_BUZZER を HIGH にして即座に警告音を鳴らす。
2. それ以外（状態0または1）の場合：
   - PIN_BUZZER を LOW にして消音状態を維持する。
```

### 3-2. millis() を使ったタイマー管理

```
【【考え方】
ステッピングモーターのパルス送信に `delay()` を使用すると、その間、超音波センサーの距離計測が完全に停止してしまい、衝突を検知できなくなります。そのため、`millis()` による差分時間管理を用い、パルス間隔が満たない間は一切処理を待たずに次のループへ処理を逃がす「ノンブロッキング制御」を実装します。


【処理の流れ】（モーター励磁パルス生成の例）
1. now = millis() で現在の時刻を取得。
2. if (now - lastMillis >= targetInterval) の条件が満たされているか確認。
3. 条件成立：次の相（A→B→C→D）に電気を流し、モーターを1ステップだけ動かす。lastMillis = now を記録。
4. 条件不成立：何もせずに関数を抜け、次の `loop()` で超音波の計測に即座に処理を回す。
```
