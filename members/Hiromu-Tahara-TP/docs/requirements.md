## 1. プロジェクト概要

### 1-1. 作品タイトル
> （例：「ものを置いたら自動でライトがつく省エネ書斎デスク」）

**タイトル：距離で色が変わるスマートインタラクティブライト**

### 1-2. 一言で言うと何をするガジェットか？
> ※ 30字以内で。AIに最初に投げるプロンプトの素になります。

**一言説明：距離で色が変わり、ボタンで通常ライトにもなる装置**

### 1-3. なぜこれを作るのか（動機・背景）
> ※ 解決したい不便さ・あったら嬉しい理由を書いてください。

**動機：手や物を近づけるだけで光が変化する“非接触操作”のライトは、業務現場の安全表示にも、家庭での間接照明にも使えると感じたため**

## 3. 機能要件

### 3-1. このガジェットができること（必須機能）
> ※ 「〜できる」の形で箇条書き。最低3つ書いてください。

- 超音波センサーで距離を測定できる
- 距離に応じてLEDの色が変化できる（30cm => 青　20cm => 黄　10cm => 赤　5cm => 赤点滅　0,5くらい）
- 距離に応じてLEDの明るさや光り方（点灯・フェード）が変化できる
- ボタンを押すと“通常ライトモード”に切り替わる
- 通常ライトモードでは、手を2cm以内に近づけると点灯／消灯が切り替わる

### 3-2. あったらいい機能（任意・余裕があれば）

- 距離の閾値を調整できる
- フェード速度を変更できるモード
- 4桁セグメントディスプレイ(本当に余裕があれば)

### 3-3. やらないこと（スコープ外）
> ※ 「〜はしない」で書いてください。AIが勝手に膨らませる機能を防ぐために重要です。

- スマホ連携はしない
- 音を出す機能はつけない
- ネットワーク通信はしない

### 3-4. 非機能要件（任意）
> ※ 余裕があれば書いてください。AIはここを無視しがちです。

- **安全性**：センサーの値が異常なとき（0や最大値など）どうするか：距離が0cmや異常値の場合はLEDを消灯し誤動作を防ぐ
- **応答時間**：操作してから何秒以内に反応すればOKか：距離変化から0.2秒以内にLEDが反応すること
- **視認性**：暗い場所でも色の違いが明確に分かる明るさであること

## 4. 使用部品リスト

### 4-1. グループ共通部品（グループ全員で使うもの・1つ）

**共通部品：超音波センサー（HC-SR04）**

**選定理由：汎用性の高さ**

### 4-2. 自分が追加で使う予定の部品（ELEGOOキットより）

| 部品名 | 何のために使うか（機能との対応） |
|:---|:---|
| 単色LED×赤・黄・青 |距離に応じて光の色・明るさを変えるため  |
| ボタン | モードを切り替えるため |
| 抵抗（220Ω ×3） | 各LEDの電流を制限し、安全に点灯させるため |
| 抵抗（10kΩ など） | ボタン入力のプルダウン／プルアップ用として安定して押下状態を検出するため |
| ジャンパー線 | Arduinoとセンサー・LED・ボタンを配線するため |
| ブレッドボード | 回路を組み立てるための土台として使うため |

## 5. AIレビュー記録

> ※ 下記の質問をAIに投げ、回答と自分の対応を記録してください。

### Q1: 不足の確認
> 「この要件定義書をもとに基本設計を作ろうとしたとき、不足している情報や曖昧な点はどこですか？」

AIの回答（要約）：LEDの種類が曖昧（単色LED3つ？RGB LED？）

                「単色LED×3」と書いてあるが、

                赤・黄・青の3色を使うのか

                どのピンに接続するのか
                が明確でない。

対応した内容： LEDは「赤・黄・青の単色LEDを使用」と明記

### Q2: 実現可能性の確認
> 「この要件はArduino UNO R3とELEGOOキットで実現できますか？難しい要件があれば教えてください。」


AIの回答（要約）：
    この要件は Arduino UNO R3 と ELEGOO キットで十分に実現可能です。
    ただし、以下の点は「工夫が必要」または「少し難易度が上がる」部分です。

    フェード機能（明るさを滑らかに変える処理）  
    → analogWrite() を使えば可能だが、明るさの変化速度やステップ数の調整が必要。

    通常ライトモードの ON/OFF（2cm以内で切り替え）  
    → 超音波センサーは近距離（2cm付近）が不安定になりやすいため、
    ノイズ対策（値の平均化や閾値の調整）が必要。

    モード切替ボタン  
    → ボタンのチャタリング対策（誤作動防止）が必要。
    また、状態管理（現在どのモードか）をコードで保持する必要がある。

    LED3色の明るさ制御（PWM）  
    → PWMピンを3つ使う必要があるため、ピン配置の設計が必要。

    距離ごとの色変化の閾値設定  
    → 超音波センサーは環境によって値がブレるため、
    10cm・20cm・30cmの境界でLEDがチラつかないように調整が必要。

対応した内容（スコープ変更など）：
    フェード機能は「簡易的な明るさ変化」に留め、複雑なアニメーションは行わない。

    通常ライトモードの ON/OFF は「2cm以内」を基準にするが、
    センサー値のブレを考慮して 閾値を微調整できるようにする。

    モード切替ボタンは 1つだけ使用し、押すたびにモードが切り替わる方式にする。

    LED は PWM 対応ピン（3・5・6）に割り当て、安定した明るさ制御を行う。

    距離の閾値（10cm・20cm・30cm）は固定値とし、
    余裕があれば後で調整できるようにする。
