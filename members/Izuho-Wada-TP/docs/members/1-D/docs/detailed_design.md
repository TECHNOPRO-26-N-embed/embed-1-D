## 🟦 `setup()`  
**役割：初期化処理**

```
・使用するピンを INPUT / OUTPUT に設定する
・シリアル通信を開始する（必要なら）
・超音波センサー、キーパッドなどのライブラリを初期化
・LED やブザーを初期状態（OFF）にする
・変数を初期値にリセットする
・起動確認のために LED を点滅させるなどの処理
```

---

## 🟦 `loop()`  
**役割：状態に応じて処理を振り分けるメインループ**

```
・現在のボタン状態を readButton() で取得し、buttonFlag を更新
・センサー値を readSensor() で取得し、distance を更新
・現在の状態 currentState に応じて以下を実行：
    - state = 0（待機）なら → doFeature1() など必要な処理
    - state = 1（スイッチ押下）なら → doFeature2() など必要な処理
    - state = 2（計測）なら → センサー値を使った処理
    - state = 3（表示）なら → 表示処理
・updateOutput() を呼び出して LED / ブザー / モーターなどを制御
```

---

## 🟦 `readButton()`  
**役割：チャタリング除去済みのボタン状態を返す**

```
・生のボタン入力を digitalRead() で取得
・前回の読み取りから一定時間（20〜50ms）経過しているか確認
・変化が安定していれば buttonFlag を更新
・確定したボタン状態（true/false）を返す
```

---

## 🟦 `readSensor()`  
**役割：超音波センサーの距離を測定して返す**

```
・trig ピンを HIGH → LOW にしてパルスを送信
・echo ピンの HIGH の長さを pulseIn() で取得し trigTime に格納
・距離 = trigTime × 0.017（cm換算）を計算
・distance に格納
・distance を返す
```

---

## 🟦 `updateOutput(state)`  
**役割：状態に応じて LED / ブザー / モーターなどを制御**

```
・state が 0（待機）なら：
    - LED OFF
    - ブザー OFF
    - プロペラ停止

・state が 1（スイッチ押下）なら：
    - LED 点灯
    - ブザー短く鳴らす

・state が 2（計測）なら：
    - 距離に応じてプロペラの PWM を調整
    - LED を点滅させるなどのフィードバック

・state が 3（表示）なら：
    - 距離をシリアルモニタや LCD に表示
```

---

```
・ボタンが押されたら状態を 1 に遷移
・必要ならタイマーをリセット
・LED やブザーのフィードバックを行う
```

## 🟦 `doFeature2()`（F02 必須機能②）  
**役割：requirements.md の必須機能②を実行**

```
・センサー値 distance を使って特定の処理を行う
・距離が一定以下ならアラートを出す
・距離に応じてプロペラ速度を変更する
・必要に応じて state を 3（表示）へ遷移
```

### `loop()` — メインループ

> ※ loop() は「状態ごとに何をするか」だけ書く。細かい処理は各関数に任せる。

```
【処理の流れ】

＜毎ループ実行すること＞
  - 入力を読む（readButton(), readSensor() などを呼ぶ）
  - 現在時刻を取得: now = millis()

＜currentState が 0（待機中）のとき＞
  - センサー値を監視する
  - 検知条件を満たしたら → currentState = 1

＜currentState が 1（動作中）のとき＞
  - メイン処理を行う
  - 終了条件を満たしたら → currentState = 2

＜currentState が 2（完了）のとき＞
  - 完了表示をする
  - リセットボタンが押されたら → currentState = 0

＜currentState が 3（エラー）のとき＞
  - エラー表示をする / リセットを待つ
```

**↓ 自分の loop() を設計してください**
```
【処理の流れ】

＜毎ループ実行すること＞
  - readButton() を呼び出してボタン状態を更新する
  - readSensor() を呼び出して距離（distance）を更新する
  - 現在時刻 now = millis() を取得する
  - 必要に応じて lastMeasureMillis などのタイマーを更新する


＜currentState が 0（待機中）のとき＞
  - センサー値 distance を監視する
  - distance がしきい値以下になるなどの検知条件を満たしたら
        → currentState = 1（動作開始）
  - LED やブザーは待機状態の表示にする（updateOutput()）


＜currentState が 1（動作中）のとき＞
  - メイン処理（doFeature1 または doFeature2）を実行する
  - 動作中のフィードバック（LED 点滅、モーター制御など）を行う
  - 終了条件（一定時間経過、距離が変化、ボタン押下など）を満たしたら
        → currentState = 2（完了）


＜currentState が 2（完了）のとき）＞
  - 完了表示（LED 点灯、LCD 表示など）を行う
  - リセットボタンが押されたら
        → currentState = 0（待機に戻る）


＜currentState が 3（エラー）のとき）＞
  - エラー表示（赤LED点灯、ブザー警告など）を行う
  - リセットボタンが押されたら
        → currentState = 0（待機に戻る）
```

## 3. 重要ロジックの詳細設計

### 3-1. チャタリング防止（デバウンス処理）

> ※ ボタンを使う場合は必ず設計してください。

```
### 3-1. チャタリング防止（デバウンス処理）

【考え方】  
スイッチを押したときに発生する高速な ON/OFF の揺れ（チャタリング）を除去し、  
「1回押したら1回だけ反応する」ようにする。

【処理の流れ】
1. 生のボタン入力 raw = digitalRead(PIN_BUTTON) を読む  
2. 現在時刻 now = millis() を取得する  
3. raw が前回の raw（lastRawState）と違う場合  
   - 状態が変化したと判断し、lastDebounceTime = now  
   - lastRawState = raw  
4. (now - lastDebounceTime) >= DEBOUNCE_DELAY の場合  
   - 状態が安定したとみなし、buttonFlag = raw  
5. buttonFlag を返す（確定したボタン状態）

【必要な変数（Section 1 に追加）】
- lastDebounceTime : unsigned long = 0  
- DEBOUNCE_DELAY : const int = 50  
- lastRawState : bool = HIGH  
- buttonFlag : bool = HIGH  
---

### 3-2. millis() を使ったタイマー管理

```
### 3-2. millis() を使ったタイマー管理

【考え方】  
delay() を使うとスイッチ入力やセンサー読み取りが止まってしまうため、  
「前回実行した時刻」を記録し、  
**今の時刻 − 前回時刻 ≥ 周期** のときだけ処理を実行する。

【処理の流れ（例：超音波センサーの定期計測）】
1. now = millis() を取得する  
2. now - lastMeasureMillis >= MEASURE_INTERVAL（例：100ms）を確認  
3. 条件を満たした場合：  
   - 超音波センサーを読み取り、distance を更新  
   - lastMeasureMillis = now  
4. 条件を満たさない場合：  
   - 何もしない（次のループで再チェック）

【自分のシステムで millis() を使う処理】

1. **超音波センサーの定期計測（100ms）**  
   - 距離を連続で測りすぎると誤差が増えるため、100msごとに計測  
   - 使用変数：lastMeasureMillis, MEASURE_INTERVAL

2. **プロペラ制御（毎ループ）**  
   - PWM 出力は delay で止めてはいけないため、常に distance に応じて更新  
   - 使用変数：distance, propellerSpeed

3. **キーパッド入力（50〜100ms）※任意機能**  
   - 入力を取りこぼさないように一定周期で読み取る  
   - 使用変数：lastKeypadMillis, KEYPAD_INTERVAL

4. **LED の状態表示（任意）**  
   - 状態に応じて点滅させる場合、millis() で周期管理  
   - 使用変数：lastLedMillis, LED_INTERVAL
```

### 3-3. その他の重要ロジック（任意）

> **【任意】** 複雑なロジックがある場合のみ記入してください。
> 例：「距離に応じたLED点灯パターン」「ゲームの衝突判定」「温度の閾値判定」

```
### 3-3. その他の重要ロジック（距離に応じたプロペラ制御）

【処理の流れ】
1. 超音波センサーで距離（distance）を取得する
2. 距離が有効範囲（例：5〜200cm）かどうかを判定する  
   - 範囲外の場合は前回の propellerSpeed を維持する
3. 距離に応じて PWM 値（0〜255）を計算する  
   - 近いほど速く、遠いほど遅くなるようにマッピングする
4. 計算した PWM 値をモータードライバに出力する
5. 必要に応じて安全制御（最大値制限・急変抑制）を行う

【入力値と出力値の関係】
- 入力：distance（cm）
- 出力：propellerSpeed（0〜255 の PWM 値）

【例：距離 → プロペラ速度のマッピング】
- 5cm → 255（最大回転）
- 50cm → 150（中速）
- 100cm → 80（低速）
- 200cm → 0（停止）

【異常値の扱い】
- distance = 0cm（センサー未検出）  
  → 前回の propellerSpeed を維持
- distance > 400cm（超音波センサーの限界）  
  → propellerSpeed = 0（安全停止）
- 距離が急激に変化した場合（±50cm以上）  
  → ノイズと判断し、前回値を使用する

```
